<Callout type="info">
	**TL;DR** - FangDash runs entirely on Cloudflare infrastructure (Workers + D1) - Three services to
	deploy: API, Web, and PartyKit (multiplayer) - `pnpm ship` deploys all three at once
</Callout>

FangDash is designed to run on Cloudflare's infrastructure. Here's how to deploy your own instance.

## What You Need

| Requirement                  | Notes                     |
| ---------------------------- | ------------------------- |
| Cloudflare account           | Free tier works           |
| Twitch Developer application | For OAuth sign-in         |
| PartyKit account             | For real-time multiplayer |

<Steps>
<Step>
## Deploy the API

The **API** runs on Cloudflare Workers with a D1 database.

1. Create a D1 database in the Cloudflare dashboard
2. Update `apps/api/wrangler.toml` with your database ID
3. Set your secrets:

```bash
bunx wrangler secret put BETTER_AUTH_SECRET --config apps/api/wrangler.toml
bunx wrangler secret put TWITCH_CLIENT_ID --config apps/api/wrangler.toml
bunx wrangler secret put TWITCH_CLIENT_SECRET --config apps/api/wrangler.toml
```

4. Deploy from the repo root:

```bash
bun ship:api
```

</Step>

<Step>
## Deploy the Web App

The **web app** deploys to Cloudflare Workers using OpenNext.

1. Update `apps/web/.env.local` with your API URL
2. Deploy:

```bash
bun ship:web
```

</Step>

<Step>
## Deploy PartyKit

**PartyKit** handles real-time multiplayer WebSocket connections.

1. Set your PartyKit token:

```bash
export PARTYKIT_TOKEN=your-token
```

2. Deploy:

```bash
bun ship:party
```

</Step>
</Steps>

## Deploy Everything at Once

```bash
bun ship
```

This runs all three deploy commands in parallel.

## CI/CD Secrets

If you're using the included GitHub Actions workflow (`.github/workflows/deploy.yml`), you need to add these secrets to your repository under **Settings → Secrets and variables → Actions**:

| Secret                   | Description                                  |
| ------------------------ | -------------------------------------------- |
| `CLOUDFLARE_API_TOKEN`   | Cloudflare API token with Workers permission |
| `BETTER_AUTH_SECRET`     | Random string used to sign auth tokens       |
| `TWITCH_CLIENT_ID`       | From your Twitch Developer application       |
| `TWITCH_CLIENT_SECRET`   | From your Twitch Developer application       |
| `PARTYKIT_LOGIN`         | PartyKit account login                       |
| `PARTYKIT_TOKEN`         | PartyKit deploy token                        |
| `PARTYKIT_HOST`          | Your PartyKit server host                    |

The deploy workflow automatically injects `BETTER_AUTH_SECRET`, `TWITCH_CLIENT_ID`, and `TWITCH_CLIENT_SECRET` as Cloudflare Worker secrets on every API deploy. The other auth-related environment variables (`BETTER_AUTH_URL`, `WEB_URL`) are plain-text vars configured in `apps/api/wrangler.toml`.

<Callout type="warn">
	If any auth secret is missing, the API will silently disable authentication — sign-in attempts will return a 503 "Auth not configured" response.
</Callout>

## Setting Up Twitch OAuth

<Steps>
	<Step>Go to the [Twitch Developer Console](https://dev.twitch.tv/console)</Step>
	<Step>Create a new application</Step>
	<Step>Set the OAuth Redirect URL to `https://your-api-domain/api/auth/callback/twitch`</Step>
	<Step>Copy the Client ID and Client Secret to your environment variables</Step>
</Steps>

<Callout type="warn">
	The Twitch OAuth redirect URL must match your deployed API domain exactly — including the path
	`/api/auth/callback/twitch`.
</Callout>
